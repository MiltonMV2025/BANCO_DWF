> **Estado:** Backend completo + Vistas completas.

## 1. Cambios en entidades JPA

### `Sucursal.java` *(nueva — `entity/`)*
Entidad nueva que representa una sucursal del banco.

| Campo | Tipo | Detalle |
|---|---|---|
| `idSucursal` | `Integer` | PK autoincremental |
| `nombre` | `String` | VARCHAR(120) |
| `direccion` | `String` | VARCHAR(255) |
| `estado` | `String` | `ACTIVO` / `INACTIVO` |
| `empleados` | `Set<Empleado>` | `@OneToMany` lazy, mappedBy="sucursal" |

### `Empleado.java` *(modificado — `entity/`)*
Se agregó la relación con `Sucursal`:

```java
@ManyToOne(fetch = FetchType.LAZY)
@JoinColumn(name = "id_sucursal", nullable = true)
private Sucursal sucursal;
```
- `nullable = true` porque un empleado puede existir sin sucursal asignada (ej. estado `EN ESPERA`).
- Se agregó índice `IX_EMPLEADO_ID_SUCURSAL` en la anotación `@Table`.

## 3. Controlador nuevo

### `GerenciaGeneralController.java` *(nuevo — `controller/`)*

```
@Controller
@RequestMapping("/gerencia/general")
@Secured("ROLE_GERENTE_GENERAL")
```

| Método | Endpoint | Descripción |
|---|---|---|
| GET | `/sucursales` | Listar sucursales con filtros opcionales |
| POST | `/sucursales` | Crear nueva sucursal (estado inicial ACTIVO) |
| POST | `/sucursales/{id}/inactivar` | Inactivar sucursal |
| GET | `/sucursales/{id}/asignar-gerente` | Vista con candidatos disponibles |
| POST | `/sucursales/{id}/asignar-gerente` | Asignar gerente (valida rol GERENTE_SUCURSAL y estado ACTIVO) |
| GET | `/acciones-personal` | Listar empleados en estado EN ESPERA |
| POST | `/acciones-personal/{id}/aprobar` | Aprobar empleado → estado ACTIVO |
| POST | `/acciones-personal/{id}/rechazar` | Rechazar empleado → estado INACTIVO |
| GET | `/movimientos` | Todos los movimientos globales con filtros |

**Reglas de negocio implementadas en el controlador:**
- Al asignar gerente: valida que el empleado tenga rol `GERENTE_SUCURSAL` y estado `ACTIVO` — tanto en el GET (filtra candidatos) como en el POST (revalida).
- Al aprobar/rechazar: valida que el empleado esté en estado `EN ESPERA` antes de cambiar el estado.

## 5. Migración de base de datos (SQL Server - Azure)

Scripts ejecutados manualmente en el servidor Azure:

```sql
-- Tabla nueva
CREATE TABLE sucursal (
    id_sucursal INT IDENTITY(1,1) NOT NULL,
    nombre      VARCHAR(120) NOT NULL,
    direccion   VARCHAR(255) NOT NULL,
    estado      VARCHAR(20)  NOT NULL,
    CONSTRAINT PK_SUCURSAL PRIMARY KEY (id_sucursal)
);
CREATE INDEX IX_SUCURSAL_ESTADO ON sucursal (estado);

-- Columna nueva en empleado
ALTER TABLE empleado ADD id_sucursal INT NULL;
CREATE INDEX IX_EMPLEADO_ID_SUCURSAL ON empleado (id_sucursal);
ALTER TABLE empleado ADD CONSTRAINT FK_EMPLEADO_SUCURSAL
    FOREIGN KEY (id_sucursal) REFERENCES sucursal (id_sucursal);
```
